# Flutter 全终端适配 + Node.js 后端 项目接口规范
本规范**前后端完全统一、全终端（Android/iOS/Web/桌面端）无差异适配**，基于 RESTful 设计 + 标准化响应体，Flutter 端可封装通用请求工具，Node.js 端可快速实现标准化接口，直接落地商用项目。

---

## 一、基础协议规范
1. **传输协议**：强制使用 `HTTPS`，全终端加密传输，杜绝安全风险
2. **环境区分**（前后端统一配置）
   | 环境     | 域名规则         | 用途          |
   | -------- | ---------------- | ------------- |
   | 开发环境 | dev-api.xxx.com  | 本地开发/联调 |
   | 测试环境 | test-api.xxx.com | 测试人员验证  |
   | 预发环境 | pre-api.xxx.com  | 上线前验收    |
   | 生产环境 | api.xxx.com      | 正式线上      |
3. **字符编码**：统一 `UTF-8`
4. **跨域处理**：Node.js 后端配置 CORS，支持 Flutter Web/移动端全终端跨域

---

## 二、接口路径与版本规范
### 1. 路径规则
`https://{域名}/api/{版本}/{资源名称}`
- 资源名称：**复数形式**，禁止动词（RESTful 核心）
- 示例：
  - 获取用户列表：`/api/v1/users`
  - 获取单用户：`/api/v1/users/123`
  - 获取文章列表：`/api/v1/articles`

### 2. 版本管理
- 路径中固定版本号：`v1、v2`，迭代不兼容接口时升级版本
- 优势：全终端兼容旧版本，无破坏性更新

---

## 三、请求方式规范（RESTful）
严格按业务语义使用，**全终端统一**：
| 请求方式 | 用途          | 示例               |
| -------- | ------------- | ------------------ |
| GET      | 查询/获取数据 | 获取用户列表、详情 |
| POST     | 新增数据      | 注册、创建文章     |
| PUT      | 全量修改数据  | 更新用户全部信息   |
| PATCH    | 局部修改数据  | 更新用户头像       |
| DELETE   | 删除数据      | 删除订单           |

---

## 四、请求规范（核心）
### 1. 统一请求头（Flutter 端自动携带）
所有接口必须携带以下请求头，适配全终端：
```json
{
  "Content-Type": "application/json; charset=utf-8", // 除文件外固定
  "Authorization": "Bearer {token}", // 鉴权令牌（登录后必填）
  "User-Agent": "flutter_android/flutter_ios/flutter_web/flutter_macos", // 终端类型
  "Device-Id": "唯一设备标识", // 全终端设备ID
  "App-Version": "1.0.0" // 客户端版本号
}
```

### 2. 请求参数规则
1. **GET 请求**：参数放在 `URL Query`（拼接在地址后）
2. **POST/PUT/PATCH/DELETE**：参数放在 `Request Body`（JSON 格式）
3. **禁止**：混合使用 Query + Body 传参

---

## 五、统一响应体规范（最核心！全终端通用解析）
**所有接口返回固定结构**，Flutter 端只需封装一个模型解析，适配所有终端：
```json
{
  "code": 200,          // 业务状态码（数字）
  "msg": "操作成功",     // 提示文案（前端直接展示）
  "data": {},           // 业务数据（对象/数组/null）
  "timestamp": 1718908800000 // 毫秒级时间戳
}
```

### 场景示例
1. 成功返回数据
```json
{
  "code": 200,
  "msg": "获取成功",
  "data": { "id": 1, "username": "test" },
  "timestamp": 1718908800000
}
```
2. 无数据返回
```json
{
  "code": 200,
  "msg": "暂无数据",
  "data": null,
  "timestamp": 1718908800000
}
```

## 六、状态码规范（前后端统一）
### 1. 通用业务状态码（`code` 字段）
| code | 含义              | 前端处理     |
| ---- | ----------------- | ------------ |
| 200  | 请求成功          | 正常解析数据 |
| 400  | 参数校验失败      | 提示 msg     |
| 401  | Token 过期/未登录 | 跳登录页     |
| 403  | 无权限访问        | 提示无权限   |
| 404  | 接口不存在        | 提示接口错误 |
| 500  | 服务器异常        | 提示系统繁忙 |

### 2. 自定义业务状态码（1000+）
用于细分业务逻辑：
- 1001：用户不存在
- 1002：密码错误
- 1003：手机号已注册
- 2001：库存不足

---

## 七、分页/排序规范（列表接口必用）
### 1. 请求参数（固定）
```json
{
  "page": 1,    // 页码，默认1
  "size": 10,   // 每页条数，默认10
  "sort": "createTime", // 排序字段（可选）
  "order": "desc"       // 排序方式：asc/desc（可选）
}
```

### 2. 响应数据（固定结构）
```json
{
  "code": 200,
  "msg": "成功",
  "data": {
    "total": 100,   // 总条数
    "pages": 10,    // 总页数
    "page": 1,      // 当前页
    "size": 10,     // 每页条数
    "list": []      // 数据列表
  },
  "timestamp": 1718908800000
}
```

## 八、鉴权认证规范（JWT，全终端通用）
1. **登录接口**：验证账号密码 → 返回 `accessToken` + `refreshToken`
2. **请求鉴权**：请求头携带 `Authorization: Bearer {accessToken}`
3. **Token 过期处理**
   - Flutter 端：自动用 `refreshToken` 刷新令牌
   - Node.js 端：统一中间件校验 Token 有效性
4. **无鉴权接口**：公开接口（如登录、注册、首页公告）直接放行

---

## 九、文件上传/下载规范
### 1. 文件上传
- 请求类型：`multipart/form-data`
- 参数：`file`（文件）+ 业务参数
- 返回：文件可访问 URL + 文件信息
```json
{
  "code": 200,
  "msg": "上传成功",
  "data": { "url": "https://xxx.com/file/123.jpg", "size": 1024 },
  "timestamp": 1718908800000
}
```

### 2. 文件下载
- 后端直接返回文件流
- Flutter 端用 Dio 统一下载，适配全终端

## 十、参数命名与数据类型规范
**前后端完全统一**，杜绝解析错误：
1. 命名：**小驼峰（camelCase）**（userId、createTime）
2. 布尔值：`true/false`，禁止用 0/1
3. 时间：毫秒级时间戳（数字类型）
4. 空值：统一返回 `null`，禁止空字符串 `""`

---

## 十一、错误处理规范
1. **后端**：统一异常中间件，所有错误返回标准响应体
2. **前端**：Dio 统一拦截器，自动处理 401/500 等错误，展示 `msg` 文案
3. **禁止**：后端直接抛出原生错误，前端无法解析

---

## 十二、安全规范
1. 参数校验：前后端双重校验，防 SQL 注入/XSS
2. 请求限流：Node.js 配置接口限流，防攻击
3. 敏感接口：日志记录操作人、终端、IP
4. Token：加密存储，过期自动失效

---

## 十三、接口文档规范
使用 **Swagger/OpenAPI** 自动生成文档：
- Node.js：`swagger-jsdoc` + `swagger-ui-express`
- 文档地址：`/api/docs`
- 包含：接口路径、参数、响应、状态码、终端适配说明

基于 `dio` 封装**全终端通用请求工具**，自动适配所有接口规范：
1. 统一添加请求头
2. 自动解析标准响应体
3. 全局错误拦截（401 跳登录、500 提示）
4. Token 自动刷新
5. 全终端（移动端/Web/桌面）无差异使用

基于 Express 封装**标准化接口工具**：
1. 统一响应工具：`success()`、`fail()` 方法
2. 全局异常中间件
3. JWT 鉴权中间件
4. CORS 全终端适配
5. 参数校验中间件
